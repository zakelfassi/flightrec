01 The Problem — Agents Are Blind Spots

YOU DEPLOYED
AN AGENT.
DO YOU KNOW
WHAT IT
CHANGED?

You give an agent access to your filesystem, your config, your codebase. It runs. It does things. Then it reports back — in natural language, optimistically, from its own perspective.

That's not observability. That's a vibes check.

Agentscope sits outside the agent. It watches what actually happened — at the filesystem layer, ground truth — and builds a timestamped, diffable, queryable record. Like git, but for runtime behavior.

02 How It Works — Watch → Diff → Narrate
STEP 01 👁
WATCH

Drop the daemon next to your agent. Point it at a directory. Agentscope subscribes to OS-level filesystem events via inotify (Linux) or FSEvents (macOS). Zero instrumentation needed on the agent side.

$ scope watch ./workspace \
  --agent=my-agent \
  --session=abc123
STEP 02 Δ
DIFF

Every event is captured: creates, modifies, deletes, permission changes, exec calls. Files are checksummed. Diffs are computed and stored in a structured event log — timestamped to microsecond precision.

event: MODIFY
path: src/config.rs
diff: +12 -3
hash: a3f9c2d…
STEP 03 ◎
NARRATE

Query the session log in natural language. Feed it to an LLM for incident review. Export as JSON, Markdown, or structured diff. The record is LLM-optimized: dense, factual, token-efficient.

$ scope narrate abc123
> Agent modified 4 files,
> deleted 1, ran 2 builds.
> Net delta: +89 -34 lines.
03 Use Cases — Where Agentscope Runs
Agentic Ops
OPENCLAW CONFIGS

OpenClaw agents touch SOUL.md, TOOLS.md, skills, cron definitions. Know exactly which agent session modified which config, when, and what changed. Full audit trail for your agent fleet.

session: noth/morning-cron
touched: skills/agent-sync/SKILL.md
delta: +2 lines, section "Ops invariants"
Code Agents
RALPH LOOP

The Ralph Loop runs Codex + Claude in a build-review cycle. Agentscope records every file the loop created, modified, or deleted — separate from the git commit. The diff before the diff.

agent: codex/gpt-5.2-codex
run: loop-iteration-47
files: 8 modified, 2 created
exec: cargo test (exit 0)
Robotics / Embedded
ANY AGENT,
ANY SYSTEM

Robots write configs. Autonomous pipelines touch ML model checkpoints. IoT agents update firmware staging directories. Agentscope works anywhere a filesystem exists. No agent SDK. No cloud dependency.

target: /opt/robot/config/
event: MODIFY calibration.yaml
actor: nav-agent-v3
hash: b2a91f… → d04c33…
